# Example Collaborative Workflow

## Topic: Database Design

1. Remember
- Instructor publishes resources in step 1.
- Student reads/watches foundational materials.

2. Understand
- Student attempts a quiz on keys, constraints, normalization.

3. Apply
- Student submits assignment applying normalization to sample data.

4. Analyze
- Student opens forum thread and debates schema trade-offs.

5. Evaluate
- Student writes a reflective blog/journal critique.

6. Create
- Students co-author wiki pages and contribute to group project artifacts.



- Instructor course designer explicitly maps each step to Bloom level + activity type.
- Bloom coverage endpoint flags missing higher-order levels.
- Dashboard analytics tracks collaborative and reflective actions, not just content access.
- Peer review and wiki features are first-class endpoints and UI actions.
